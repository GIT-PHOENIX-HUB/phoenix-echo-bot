## What's New in v1.0

### 1. Enhanced Authentication

**Environment Variable:**
```bash
export PHOENIX_GATEWAY_TOKEN="your-secret-token-here"
```

**Config File (`~/.phoenix-echo/config.json`):**
```json
{
  "gateway": {
    "auth": {
      "mode": "token",
      "token": "your-secret-token-here"
    }
  }
}
```

**Usage:**
- All `/api/*` endpoints require authentication
- WebSocket `/ws` endpoint requires authentication
- `/health` endpoint is public (no auth required)

**Passing Token:**
- Header: `X-Phoenix-Token: your-token`
- Bearer: `Authorization: Bearer your-token`
- Query: `?token=your-token`

**Test:**
```bash
# Without token (should fail)
curl http://localhost:18790/api/sessions
# {"error":"Unauthorized: invalid gateway token"}

# With token (should work)
curl -H "X-Phoenix-Token: your-token" http://localhost:18790/api/sessions
```

### 2. Structured Logging

**Log Format (JSON):**
```json
{
  "timestamp": "2026-02-21T05:16:32.043Z",
  "level": "INFO",
  "component": "gateway",
  "message": "Phoenix Echo Gateway starting",
  "version": "1.0.0",
  "workspace": "/path/to/workspace"
}
```

**Log Levels:**
- `DEBUG` - Detailed debugging information
- `INFO` - General informational messages
- `WARN` - Warning messages (non-critical issues)
- `ERROR` - Error messages (failures)
- `CRITICAL` - Critical issues (system failures)

**Configuration:**
```json
{
  "logging": {
    "level": "info",
    "file": "/tmp/phoenix-echo/phoenix-echo.log"
  }
}
```

**Environment Variables:**
```bash
export PHOENIX_LOG_LEVEL="debug"
export PHOENIX_LOG_FILE="/var/log/phoenix-echo/phoenix.log"
```

**Features:**
- Daily log rotation (automatic)
- 7-day retention (configurable)
- Correlation IDs on all requests (`x-request-id`)
- Dual output: stdout + file

**Log File Location:**
- Default: `/tmp/phoenix-echo/phoenix-echo-YYYY-MM-DD.log`
- Rotates daily at midnight
- Old logs automatically cleaned up

### 3. Config Hot-Reload

**Config File Location:**
`~/.phoenix-echo/config.json`

**Override with Environment Variable:**
```bash
export PHOENIX_CONFIG_PATH="/custom/path/to/config.json"
```

**How It Works:**
1. Edit `~/.phoenix-echo/config.json`
2. Save the file
3. Gateway automatically reloads (500ms debounce)
4. New config applied without restart

**What Can Be Hot-Reloaded:**
- ✅ Log level
- ✅ Log file path
- ❌ Port (requires restart)
- ❌ Workspace (requires restart)

**Monitoring Reloads:**
Watch logs for:
```json
{
  "level": "INFO",
  "message": "Config file changed, reloading...",
  "configPath": "~/.phoenix-echo/config.json"
}
{
  "level": "INFO",
  "message": "Configuration reloaded successfully",
  "port": 18790,
  "logLevel": "debug"
}
```

### 4. Graceful Shutdown

**Shutdown Sequence:**
1. Stop accepting new requests
2. Snapshot all active sessions
3. Close WebSocket clients (3s grace period)
4. Close HTTP server (3s grace period)
5. Stop config file watcher
6. Exit cleanly

**Triggering Shutdown:**
```bash
# Send SIGTERM (graceful)
kill -TERM <pid>

# Send SIGINT (Ctrl+C)
kill -INT <pid>

# Or in terminal
Ctrl+C
```

**Forced Exit:**
- If shutdown takes >6s, process force-exits
- Prevents hanging during shutdown

**Log Output:**
```json
{"level":"WARN","message":"shutdown_start","signal":"SIGTERM"}
{"level":"INFO","message":"shutdown_snapshots_complete","count":3}
{"level":"INFO","message":"shutdown_ws_clients_closed"}
{"level":"INFO","message":"shutdown_http_closed"}
{"level":"INFO","message":"shutdown_config_watcher_stopped"}
```

### 5. Session Management API

#### **List All Sessions**
```bash
GET /api/sessions

Response:
{
  "sessions": [
    {
      "sessionId": "main",
      "messageCount": 150,
      "updatedAt": "2026-02-21T05:00:00Z",
      "lastRole": "assistant",
      "preview": "Here are the results..."
    }
  ],
  "requestId": "uuid"
}
```

#### **Get Session Details**
```bash
GET /api/sessions/:sessionId

Response:
{
  "sessionId": "main",
  "messages": [...],
  "checkpoint": {...},
  "requestId": "uuid"
}
```

#### **Compact Session**
```bash
POST /api/sessions/:sessionId/compact
Content-Type: application/json

{
  "strategy": "keep-recent",
  "keepCount": 50
}

Response:
{
  "sessionId": "main",
  "strategy": "keep-recent",
  "originalCount": 150,
  "finalCount": 50,
  "originalTokens": 45000,
  "finalTokens": 15000,
  "compacted": true,
  "tokensSaved": 30000,
  "compressionRatio": "66.7%",
  "requestId": "uuid"
}
```

**Compaction Strategies:**

1. **keep-recent** (default)
   - Keep last N messages
   - Summarize everything else
   - Good for: Manual cleanup

2. **token-limit**
   - Compact until under token threshold
   - Removes oldest 25% at a time
   - Good for: Automatic context management

**Usage:**
```bash
# Keep last 50 messages
curl -X POST http://localhost:18790/api/sessions/main/compact \
  -H "Content-Type: application/json" \
  -H "X-Phoenix-Token: your-token" \
  -d '{"strategy":"keep-recent","keepCount":50}'

# Compact to under 100K tokens
curl -X POST http://localhost:18790/api/sessions/main/compact \
  -H "Content-Type: application/json" \
  -H "X-Phoenix-Token: your-token" \
  -d '{"strategy":"token-limit","tokenLimit":100000}'
```

```bash
# Gateway
export PHOENIX_PORT=18790
export PHOENIX_BIND="127.0.0.1"
export PHOENIX_GATEWAY_TOKEN="your-secret-token"

# Workspace
export PHOENIX_WORKSPACE="/path/to/workspace"

# Config
export PHOENIX_CONFIG_PATH="/custom/config.json"
```

## Production Deployment Checklist

### Pre-Deployment

- [ ] Create `~/.phoenix-echo/config.json`
- [ ] Set `PHOENIX_GATEWAY_TOKEN` (generate secure token)
- [ ] Configure `ANTHROPIC_AUTH_TOKEN` or auth profile
- [ ] Set workspace path (`PHOENIX_WORKSPACE`)
- [ ] Configure log file location
- [ ] Test config hot-reload
- [ ] Test authentication

### LaunchDaemon Setup (macOS)

Create `/Library/LaunchDaemons/ai.phoenixelectric.gateway.plist`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>ai.phoenixelectric.gateway</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/local/bin/node</string>
        <string>/path/to/phoenix-echo-gateway/src/index.js</string>
    </array>
    <key>WorkingDirectory</key>
    <string>/path/to/phoenix-echo-gateway</string>
    <key>EnvironmentVariables</key>
    <dict>
        <key>PHOENIX_GATEWAY_TOKEN</key>
        <string>your-secret-token</string>
        <key>ANTHROPIC_AUTH_TOKEN</key>
        <string>your-oauth-token</string>
        <key>PHOENIX_WORKSPACE</key>
        <string>/var/phoenix/workspace</string>
        <key>PHOENIX_LOG_FILE</key>
        <string>/var/log/phoenix-echo/phoenix.log</string>
    </dict>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
    <key>StandardOutPath</key>
    <string>/var/log/phoenix-echo/stdout.log</string>
    <key>StandardErrorPath</key>
    <string>/var/log/phoenix-echo/stderr.log</string>

</plist>
```

Load:
```bash
sudo launchctl load /Library/LaunchDaemons/ai.phoenixelectric.gateway.plist
```

## Troubleshooting

### Issue: Server won't start

**Check:**
1. Port already in use? `lsof -i :18790`
2. Config file syntax error? `cat ~/.phoenix-echo/config.json | jq .`
3. Missing auth credentials? Check env vars or auth-profiles.json

### Issue: Authentication failing

**Check:**
1. Token set? `echo $PHOENIX_GATEWAY_TOKEN`
2. Token in config? `cat ~/.phoenix-echo/config.json | jq .gateway.auth.token`
3. Correct header? Use `X-Phoenix-Token` or `Authorization: Bearer`

### Issue: Logs not appearing in file

**Check:**
1. Directory exists? `ls -la /tmp/phoenix-echo/`
2. Write permissions? `touch /tmp/phoenix-echo/test.log`
3. Log level too high? Set to `debug` to see all messages

### Issue: Config hot-reload not working

**Check:**
1. Config file exists? `ls -la ~/.phoenix-echo/config.json`
2. File watcher errors in logs? Search for `config_watcher_error`
3. Changes saved? File must be written to disk, not just edited

### Issue: Sessions not compacting

**Check:**
1. Session has enough messages? Need >keepCount messages
2. Strategy correct? Use `keep-recent` or `token-limit`
3. Session ID valid? Check `GET /api/sessions` for exact IDs

## API Reference Summary

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/health` | GET | No | Health check |
| `/api/recovery` | GET | Yes | Recovery details |
| `/api/sessions` | GET | Yes | List all sessions |
| `/api/sessions/:id` | GET | Yes | Get session details |
| `/api/sessions` | POST | Yes | Create session |
| `/api/sessions/rename` | POST | Yes | Rename session |
| `/api/sessions/:id/compact` | POST | Yes | Compact session |
| `/api/chat` | POST | Yes | Send message (REST) |
| `/ws` | WS | Yes | WebSocket endpoint |

**All authenticated endpoints require:**
- Header: `X-Phoenix-Token: <token>`
- OR Bearer: `Authorization: Bearer <token>`
- OR Query: `?token=<token>`

**All responses include:**
- Correlation ID: `requestId` field
- HTTP header: `x-request-id`
